## 🔄 РАБОЧИЙ ПРОЦЕСС (Workflow)

### Этап 1: Разработка (Qwen Code)

```
1. Менеджер ставит задачу → Qwen Code
2. Qwen Code пишет/исправляет код (локально, на MacBook)
3. Qwen Code создаёт инструкцию для Agent
4. Qwen Code НЕ делает git commit/push (это делает Agent)
```

**Результат:** Код готов локально, инструкция создана

---

### Этап 2: Инструкция для Agent (Qwen Code)

```
1. Qwen Code создаёт инструкцию
2. Присваивает версию протокола (X.Y.0)
3. Сохраняет в exchange/toopencode.md (локально)
4. Менеджер отправляет на сервер через SCP
   → /var/www/tarot-miniapp/exchange/toopencode.md
```

**Содержание инструкции:**
- Версия протокола (X.Y.0)
- Приоритет (КРИТИЧЕСКИЙ/ВЫСОКИЙ/СРЕДНИЙ)
- Задача (что сделать)
- Команды для выполнения (git pull, pm2 restart, etc.)
- Проверки (что проверить после деплоя)
- Срок выполнения
- **Разрешение на пуш** (если требуется для кода)

---

### Этап 3: Коммит и пуш кода (Opencode Agent)

```
1. Opencode Agent читает инструкцию из exchange/toopencode.md
2. Если код уже написан Qwen → делает git pull
3. Если код требует изменений → Qwen пишет, Agent пушит
4. Opencode Agent делает коммит и пуш (с разрешения для кода)
```

**Для кода (требуется разрешение):**
```bash
cd /var/www/tarot-miniapp
git add -A
git commit -m "[тип]: описание изменений

Co-authored-by: Qwen-Coder <qwen-coder@alibabacloud.com>"
git push origin main
```

**Для документации/отчётов (без разрешения):**
```bash
cd /var/www/tarot-miniapp
git add exchange/toQwen.md
git commit -m "docs: отчёт по задаче [название]"
git push origin main
```

**Результат:** Код/отчёт на GitHub в ветке `main`

---

### Этап 4: Деплой (Opencode Agent)

```
1. Opencode Agent читает инструкцию из exchange/
2. Выполняет git pull origin main
3. Перезапускает сервисы (pm2 restart)
4. Проверяет статус (pm2 status, curl API)
```

**Результат:** Код на сервере обновлён

---

### Этап 5: Тестирование (Opencode Agent)

```
1. Opencode Agent выполняет тесты из инструкции
2. Проверяет логи (pm2 logs)
3. Проверяет API (curl)
4. Проверяет функционал (ручное тестирование)
5. Записывает результаты в отчёт
```

---

### Этап 6: Отчёт (Opencode Agent)

```
1. Opencode Agent пишет отчёт
2. Сохраняет в exchange/toQwen.md (на сервере)
3. Opencode Agent пушит отчёт на GitHub
4. Qwen Code читает отчёт из GitHub (локально, через git pull)
```

**Формат отчёта:**
```markdown
# Отчёт Opencode Agent

## Задача: [название]

## Выполнено:
- [ ] пункт 1
- [ ] пункт 2

## Тесты:
- Тест 1: ✅/❌
- Тест 2: ✅/❌

## Проблемы:
1. [описание]

## Логи:
[ключевые сообщения]

## Версия протокола: X.Y.1
## Статус: [ГОТОВО / ТРЕБУЕТ ИСПРАВЛЕНИЯ]
```

**🚨 ОТПРАВКА ОТЧЁТА НА GITHUB**

Opencode Agent **ОБЯЗАН** запушить отчёт (это документация, разрешение НЕ требуется):

```bash
cd /var/www/tarot-miniapp
git add exchange/toQwen.md
git commit -m "docs: отчёт по задаче [название]"
git push origin main
```

**Зачем пушить отчёт:**
- ✅ Qwen Code читает отчёт локально (через `git pull`)
- ✅ История отчётов сохраняется в Git
- ✅ Менеджер видит прогресс
- ✅ Нет задержек на SCP

### Этап 7: Ревью и исправление (Qwen Code)

```
1. Qwen Code делает `git pull origin main`
2. Читает отчёт из `exchange/toQwen.md` (локально)
```

## 📊 СИСТЕМА ВЕРСИОНИРОВАНИЯ ПРОТОКОЛА

### **Формат версии:** `X.Y.Z`

**Где:**
- **X (первая цифра)** — мажорная версия (изменения в протоколе)
- **Y (вторая цифра)** — номер задачи/инструкции (растёт с каждой инструкцией)
- **Z (третья цифра)** — статус выполнения:
  - **0** — инструкция отправлена (Qwen)
  - **1** — инструкция выполнена (Agent)

**Текущая версия:** 6.0.0 (сводная)

---

### **Правила версионирования:**

#### **Qwen Code:**

1. **Отправляет инструкцию:** Версия `X.Y.0`
2. **Ждёт отчёт от Agent:** Версия `X.Y.1`
3. **Новая инструкция:** Версия `X.(Y+1).0`

**Пример:**
```
Инструкция "Админка" → Версия 5.1.0
Отчёт "Админка" → Версия 5.1.1

Инструкция "Оплата" → Версия 5.2.0
Отчёт "Оплата" → Версия 5.2.1
```

---

#### **Opencode Agent:**

**В отчёте указывает:**

1. **Если инструкция НЕ выполнена:**
   ```markdown
   **Версия протокола:** 5.1.0
   **Статус:** ⏳ В ПРОЦЕССЕ / ❌ НЕ ВЫПОЛНЕНО
   ```

2. **Если инструкция ВЫПОЛНЕНА:**
   ```markdown
   **Версия протокола:** 5.1.1
   **Статус:** ✅ ВЫПОЛНЕНО
   ```

---

### **БЛОКИРОВКА НОВЫХ ИНСТРУКЦИЙ:**

**Qwen Code НЕ может дать новую инструкцию пока:**

```
❌ Opencode Agent не отчитался с версией X.Y.1
❌ Статус не "ВЫПОЛНЕНО"
❌ Менеджер не дал разрешение на нарушение
```

**Пример:**

```
Qwen Code отправил: Версия 5.1.0
Opencode Agent должен ответить: Версия 5.1.1

ТОЛЬКО ПОСЛЕ ЭТОГО:
Qwen Code может отправить: Версия 5.2.0
```

---

### **ИСКЛЮЧЕНИЕ (нарушение протокола):**

**ТОЛЬКО менеджер может разрешить нарушение:**

```
МЕНЕДЖЕР: "Разрешаю нарушение протокола"
Qwen Code: Может отправить новую инструкцию без получения X.Y.1
```

**Причина нарушения должна быть задокументирована:**

```markdown
**Нарушение протокола:**
- Разрешено менеджером: ДА
- Причина: [критический баг, срочный деплой, etc.]
- Версия: 5.1.0 → 5.2.0 (без получения 5.1.1)
```

## 📊 ПРИМЕР ЦИКЛА РАЗРАБОТКИ

### Задача: Исправить кнопки в админке

**1. Qwen Code пишет исправление:**
```bash
git add -A
git commit -m "fix: кнопки не работали"
git push origin main
```

**2. Qwen Code создаёт инструкцию:**
```bash
cat > /tmp/toOpencode_buttonfix.md << 'EOF'
# Исправление кнопок

## Деплой:
git pull origin main
pm2 restart tarot-server

## Тесты:
1. Проверить кнопки "Тарологи"
2. Проверить кнопки "Транзакции"
3. Проверить модальные окна

## Отчёт:
exchange/toQwen.md
EOF

scp /tmp/toOpencode_buttonfix.md root@server:/var/www/tarot-miniapp/exchange/
```

**3. Opencode Agent выполняет:**
```bash
cd /var/www/tarot-miniapp
git pull origin main
pm2 restart tarot-server

# Тесты
# ...

# Отчёт
cat > exchange/toQwen.md << 'EOF'
# Отчёт

## Тесты:
- Кнопки "Тарологи": ✅
- Кнопки "Транзакции": ✅
- Модальные окна: ✅

## Статус: ГОТОВО
EOF

git add exchange/toQwen.md
git commit -m "docs: отчёт - исправление кнопок"
git push origin main
```

**4. Qwen Code читает отчёт:**
```bash
git pull origin main
cat exchange/toQwen.md
```

**5. Если статус "ГОТОВО" → задача закрыта**
**Если статус "ТРЕБУЕТ ИСПРАВЛЕНИЯ" → повторить цикл**
